Project Overview

Simulating a Business Network Inside the Home Lab

Inside the home lab, I've set out to simulate key components of a real-world business network. This includes building a structured environment with domain controllers, client machines, segmented networks, and logging infrastructure — closely mimicking how enterprise networks are designed and secured.

Provisioned Systems
  • Domain Controller Centralized authentication and directory services (Active Directory)
  • SMTP Relay Server Simulates internal mail flow for phishing, alert testing, and email logging
  • Dedicated Security Server Hosts tools like ELK Stack, Wazuh, and Suricata for log analysis and network monitoring
  • Security Playground An intentionally vulnerable VM used to simulate misconfigurations, exploits, and malware behavior
  • Windows Workstation A domain-joined client machine for endpoint testing and user simulation
  • Linux Desktop Workstation Used for analysis, scripting, and multi-platform behavior comparison
  • Attacker Environment A Kali Linux system loaded with offensive tools to simulate adversary behavior (e.g., reconnaissance, exploitation, lateral movement)

This lab allows me to safely experiment with system hardening, detection engineering, threat emulation, and incident response from both Red Team and Blue Team perspectives — all in a controlled environment.

VirtualBox Configuration Steps

Step 1: Setting Up a NAT Network

Created a NAT Network in VirtualBox to allow internal VM communication with controlled internet access:

  • Navigated to File ➔ Tools ➔ Network Manager
  • Created new NAT Network named "project-x-network"
  • Configured IPv4 prefix according to project specifications
Step 2: ISO Files

Downloaded official Microsoft evaluation ISOs:

  • Windows Server 2022
  • Windows 11
Step 3: Virtual Machine Creation

VM Specifications:

  • RAM: 4096 MB (4 GB)
  • CPU: 2 Cores minimum
  • Storage: 50 GB VHD
  • EFI: Enabled for Windows 11
Step 4-5: VM Configuration
  • Mounted ISO files to virtual optical drive
  • Connected VMs to project-x-network
  • Configured network settings for internal communication
Step 6-8: System State Management
  • Completed OS installation
  • Created "Post-OS Install – Clean State" snapshot
  • Documented restoration procedures
